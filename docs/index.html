How it works
Stage. Press. Done.
01

Stage your files

Run git add like you normally would. Commitr reads exactly what is staged and nothing more.

02

Press the button

Click the icon in the Source Control panel or press the keyboard shortcut. Commitr sends your diff to a local model via Ollama.

03

Watch it commit

The commit runs in your terminal automatically. Clean, specific, conventional. Your git history stays readable.

Setup

Two commands. One time.

Install Ollama, pull the model, and you are done. No accounts. No billing. No configuration required. Commitr finds your local Ollama automatically.

Download Ollama
terminal
# pull the model once
$ollama pull qwen2.5:3b

# start ollama
$ollama serve

# that's it.

What you get
privacy

Fully local

Your diff runs through Ollama on your machine. Nothing is sent anywhere, ever.

speed

Fast generation

qwen2.5:3b is small and quick. Most messages generate in under 3 seconds.

format

Conventional Commits

Follows the spec by default. Switch to freeform if you prefer unstructured output.

smart

Scope inference

Reads your file paths and picks the right scope. Override with custom mappings per project.

teams

Team ready

Solo or team

Solo dev

Just install and go.

Commitr picks up your VS Code settings and works out of the box. Configure once under Settings and forget it. No files to commit, no setup to share.

Team

One file. Everyone aligned.

Add a .commitrrc to your project root and commit it. Every teammate automatically gets the same model, style, and scope rules. No one touches their settings manually.

{
  "model": "qwen2.5:3b",
  "commitStyle": "conventional",
  "scopeMappings": {
    "src/auth": "auth"
  }
}
